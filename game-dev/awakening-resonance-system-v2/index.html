一句话总结

背包不是棋盘。你管理的不是最优解，是一群越来越吵的室友。你越来越不确定——到底是你拥有它们，还是它们正在习惯你。

设计来源：山屋惊魂的"不可知的临界点"

《山屋惊魂》最精妙的设计不是"叛徒机制"，而是临界点的不可知性。

每张预兆牌翻开时，玩家掷6颗骰子。如果点数小于场上预兆牌数量，“作祟"触发。问题是：场上已经有4张预兆牌时，你翻开第5张——掷骰子。你心里算：6颗骰子平均期望是21点，减5张牌是16，远大于4……但你还是可能掷出3。2%的概率。而它可能就在这一刻发生。

你知道它越来越近了。但你不知道是哪一张牌、哪一次掷骰、哪一步跨过门槛。

这种"不可知但可感知"的张力，是觉醒共鸣系统的核心锚点。我们不做精确的数值计量，不做"距离临界点还差23点"的UI。我们让玩家通过行为来感知——物品在动、描述在变、世界在回应——但永远不知道"下一次长休会不会就是那一刻”。

核心原则：四不暴露

不暴露 原因 玩家如何感知
不暴露共鸣值 数值是安全的。安全消除恐惧 通过物品行为、环境异常、身体变化
不暴露物品属性 饥饿度、共鸣度是设计者的语言，不是玩家的 通过观察：“它总是滚向左边”、“它碰到铭牌时会发光”
不暴露临界点 知道阈值=可以控制=可以优化=没有恐惧 通过"异常事件"的频率感知"越来越近了"
不暴露因果关系 明确的因果=合同=没有暧昧=没有叙事张力 玩家自己推断：“上次它靠近铭牌后，我受伤时好像没那么痛”

核心机制一：背包拓扑学（保留）

背包是二维网格。这一点不变——位置有意义。

但这不是策略棋盘。我们不鼓励玩家"为了最优共鸣而排列"。网格的存在是为了让物品有地方去——让它们的移动、相邻、分离都成为可见的叙事行为。

规则：

  1. 觉醒物品各自占据一格，不堆叠
  2. 普通物品可堆叠，不触发相邻行为
  3. 相邻 = 上下左右
  4. 玩家可以手动移动觉醒物品，但移动后该物品进入"休眠"——若干次长休内不再自主移动（玩家干预是有代价的：你强迫了它，它暂时失去活力）

核心机制二：觉醒物品——不暴露属性，只暴露行为

没有"饥饿度"、没有"共鸣度"的UI

玩家打开背包，看不到任何数字。看不到"饥饿度：7/10"。看不到"共鸣度：5"。

玩家看到的只有：行为。

物品会做什么（暴露的行为）

行为 玩家观察 含义（玩家自己推断）
滚动 长休后，物品在背包里移动了一格 “它想靠近/远离什么”
脉动 物品边缘有微弱的光效，频率不稳定 “它很活跃"或"它很紧张”
变色 物品图标的色调微微偏移（偏暖/偏冷） “它喜欢/不喜欢现在的邻居”
低语 查看物品时，描述文字偶尔变化 “它想告诉我什么”
发热 战斗中使用该物品时，屏幕边缘有热扭曲 “它很在乎这场战斗”
沉默 物品连续多次长休不移动，光效消失 “它休眠了"或"它不高兴了”

玩家如何"理解"一件物品

没有属性面板。玩家通过记录和观察来推断：

「晶核已经三次长休后朝左边滚动了。左边是铭牌。它喜欢铭牌？还是铭牌吸引了它？上次它碰到铭牌后，铭牌上的名字短暂显示了一秒。这是……好事吗？」

玩家会形成私人假设。这些假设不一定对——但玩家会据此行动。而行动会产生后果。而后果会模糊地、暧昧地反馈给玩家。

这就是认知的代价：理解 = 异化。你以为你理解了它，但也许只是它让你以为你理解了它。

核心机制三：异常事件系统——替代数值阈值

删除的内容

  • 共鸣值 20/40/60/80/100 的精确阈值表
  • 总共鸣值的计算公式
  • “距离 Level 3 还有 12 点"的UI提示

新增的内容：异常事件（Anomaly）

异常事件是玩家可以直接观察到的"不对劲”。它们不精确、不累积成一个进度条。它们只是发生，然后留下痕迹。

异常事件的类型

类型 触发条件 玩家观察 频率
物品移动 长休后，觉醒物品移动一格 打开背包，发现位置变了 低频→高频（随着游戏推进）
描述变化 查看觉醒物品时，文字有概率不同 同一件物品，不同时间看，描述不一样 始终低概率，但随进程增加
战斗异常 战斗日志中出现不属于战斗的文本 “你打了它。它知道是你打的。” 极低频→中频
环境异常 场景中出现之前没有的东西 墙壁上多了一条菌丝脉络，但不确定是新出现的还是之前没注意到 极低频
身体异常 玩家角色在非战斗状态下出现变化 底座阴影闪烁、视野边缘短暂模糊 极低频→低频
NPC异常 NPC对话中出现不属于其记忆的内容 一个普通商人突然说：“你背包里的那个……它还好吗？” 极低频

异常事件不累积成数值，但玩家会"感觉到"

没有"异常事件计数器：17/50"。没有进度条。

但玩家会主观地感受到：

  • 早期游戏：“偶尔会有些奇怪的事，但很少”
  • 中期游戏：“每次长休都要检查背包，有些东西在动”
  • 后期游戏：“战斗日志里那些不属于战斗的文本越来越频繁了”

这是主观的、不可靠的感知。 玩家可能夸大，可能忽略。但这正是我们想要的——“认知的代价"意味着你不再确定你的感知是准确的。

觉醒事件的触发：不是"达到80”，而是"某一次异常事件的质变"

觉醒事件（山屋惊魂的"作祟"）不是由精确数值触发的，而是由**异常事件的"质变"**触发的。

具体设计：

  • 游戏后台有一个觉醒事件候选池。每次异常事件发生时，该池获得一个"候选值"（这个值玩家不可见，设计者也不需要在UI中暴露）。
  • 当候选池积累到一定程度，下一次异常事件会自动升级为觉醒事件——但不是确定的。它可能在第5次异常事件时发生，也可能在第20次。设计者可以调整概率曲线，让它在大多数玩家的第10-15次异常事件时发生。
  • 关键：玩家永远无法知道"下一次会不会是觉醒事件"。就像山屋惊魂中翻开预兆牌后的掷骰——你知道概率在增加，但你不知道这一次会不会触发。

觉醒事件的演出：

  1. 一次异常事件发生时（例如：长休后打开背包，发现物品移动），但这次不是普通的移动——
  2. 屏幕剧烈震动
  3. 所有觉醒物品同时发光
  4. 玩家听到/看到：「你听到了。我们等你很久了。」
  5. 从这一刻起，物品的"请求"开始出现，游戏规则进入第二阶段

核心机制四：含混请求——替代明确的条件-奖励

删除的内容

  • “晶核请求：靠近铭牌。满足后：治疗效果+100%"
  • “菌丝藤蔓请求：处于中心。满足后：攻击附加中毒”
  • 明确的条件-奖励结构

新增的内容：含混请求（Oblique Request）

物品的"请求"不是以任务形式出现的。它们是以行为、暗示、暧昧的文本出现的。玩家需要猜测它们想要什么，而猜测本身可能就是一种异化。

请求的表现形式

形式 例子 玩家的解读困境
物理行为 晶核在背包里朝铭牌方向滚了一下，碰到后铭牌上的名字短暂显示了一秒 “它想要铭牌？还是它在回应铭牌？如果我让它们分开，铭牌还会发光吗？”
描述暗示 物品描述从"它在你手中脉动"变成"它在你手中发冷。它想念热的东西” “热的东西？火把？另一个脉动物品？我的体温？”
环境回应 战斗中使用该物品后，场景中的菌丝脉络短暂变亮 “它喜欢战斗？还是它喜欢菌丝？还是两者只是巧合？”
身体反馈 玩家带着这件物品进入某个区域时，屏幕边缘有微弱的温暖感 “它在指引我去哪里？还是我自己在想象？”
NPC提及 某个NPC看到玩家的背包时说：“你……把它们放得很近。你确定吗？” “很近？哪些？为什么不确定？”

因果是暧昧的，不是合同式的

传统设计（我们要避免的）：

条件：晶核与铭牌相邻。结果：治疗效果+100%。玩家看到buff图标。明确、可预测、可优化。

我们的设计：

晶核碰到铭牌后，铭牌上的名字短暂显示。下次战斗中，玩家受到伤害时，晶核微微发热——伤害似乎减轻了一点。但玩家不确定：这是晶核的效果？还是铭牌的？还是它们在一起的某种"化学反应"？还是只是那次战斗的随机波动？

玩家会自己建立因果假设。 但这些假设：

  • 可能对了
  • 可能错了
  • 可能部分对了但理由错了
  • 可能对的因果在不同时间效果不同（因为物品也在"变化"）

请求的"满足"与"不满足"不显示为明确的奖励/惩罚

玩家行为 物品的可能回应 玩家感知
让晶核靠近铭牌 晶核脉动频率增加；战斗中偶尔有微弱热感 “它们好像……更开心了？”
让晶核远离铭牌 晶核进入沉默状态；描述变成"它在你手中发冷" “它好像……不高兴了？”
晶核和另一个觉醒物品相邻 两者都脉动，但频率不协调；屏幕边缘有轻微闪烁 “它们在……打架？还是在聊天？”
晶核独处 晶核缓慢、稳定地脉动；没有异常事件 “它好像……睡着了？”

没有明确的+100%或-50%。 只有模糊的感觉：“它好像更喜欢这样"或"它好像不喜欢那样”。玩家会据此调整，但永远不会100%确定自己的调整是"正确"的。

核心机制五：同化度——隐藏数值，暴露身体

同化度不在UI中显示

玩家看不到"同化度：47%"。没有任何进度条、数字、百分比。

玩家看到的是身体的变化——分阶段出现，不可逆转。

同化阶段（通过身体变化感知）

阶段 身体变化 玩家如何感知
** untouched** 无 正常。Demo阶段
接触 长休时偶尔看到不属于自己记忆的闪回（画面片段，0.5秒） “刚才那是什么？……算了，可能是bug。”
渗透 底座阴影偶尔在静止时轻微抖动；视野边缘有极淡的菌丝纹理 “我注意到我的角色影子不太对……但好像只是偶尔。”
融合 能听到"耳语"——战斗日志中的异常文本；某些NPC会避开你 “那个商人以前会说话，现在只看我一眼就转身走了。”
节点 能看到所有隐藏的菌丝网络节点；普通治疗对你效果减半；存读档时位置可能偏移 “我能看到……这些蓝色的路径？它们一直在那里吗？”

同化度增长的来源（对玩家隐藏，设计者可见）

事件 设计者端的影响 玩家端的感知
每次觉醒事件 大幅推进 “那次事件之后，一切都变了”
每次使用觉醒物品 小幅推进 无明显感知，但长期使用后"身体变化"会累积
每次长休携带觉醒物品 小幅推进 同上
进入菌丝网络节点 大幅推进 进入特定区域后，感觉"更近了"
与同化NPC互动 中幅推进 与某些NPC对话后，感觉"有什么东西传过来了"

设计者可以通过调整这些事件的频率和幅度，来控制同化度增长的速度。 但玩家始终只是通过"身体变化"来感知，没有数字。

核心机制六：觉醒事件后的世界

觉醒事件（山屋惊魂的"作祟触发"）发生后，游戏不会弹出一个教程说"现在规则改变了"。规则改变是通过行为来体现的。

觉醒事件后的变化（玩家自己发现）

变化前 变化后 玩家如何发现
觉醒物品偶尔移动 觉醒物品每次长休都移动，且移动路径更明确 打开背包，发现它们的位置越来越有"意图"
物品描述偶尔变化 物品描述频繁变化，且开始出现"第二人称"——物品直接对你说话 查看物品时，描述变成「你为什么让我离它那么远？」
战斗中偶尔有异常文本 战斗日志中频繁出现不属于战斗的文本，且开始出现"指导"——告诉你敌人的弱点 “它怕热。你手里有热的吗？"——玩家不确认这是否真的，但如果尝试使用火系攻击，发现确实有效……
环境偶尔有异常 环境中频繁出现新的菌丝路径，且这些路径在地图上形成"网络” 探索时发现自己能看到之前看不到的路径，通往隐藏区域
物品没有明确请求 物品开始主动靠近你关心的事物——例如：在战斗前，晶核滚向你的武器 “它好像……知道我要战斗？”

同伴的变化

觉醒事件后，同伴（如果有）会开始"听到"一些东西：

  • 同伴在战斗前突然说：“它说……从左边攻击。”
  • 同伴在探索时说：“你有没有觉得……墙壁在呼吸？”
  • 同伴看你的眼神变了：不是不信任，是不确定——不确定你还是不是"你"。

Demo中的行为（玩家可能注意到，也可能错过）：

  • 晶核在背包中不堆叠
  • 如果玩家同时有"孢子平台感知buff"（一件微弱的觉醒物品），晶核在Demo中会自动朝它靠近一格（如果玩家长休过——但Demo中只有一次长休机会，在Boss战后，所以大多数玩家不会看到这个行为）
  • 打开背包时，晶核边缘有微弱的脉动——大多数玩家会以为是"发光道具的普通效果"
  • 如果玩家多次查看晶核的描述（在Demo中不太可能，但二周目可能），有低概率看到变异版本

种子2：铭牌计数器——异常事件的视觉语言预演

Demo中的铭牌计数器视觉污染（11→12→11）是异常事件系统的视觉语言：

  • 数字"说谎" = 物品/系统在"自主行动"
  • 快速一闪（回声裂隙）vs 缓慢呼吸（主线）= 不同的"行为模式"
  • 玩家可能觉得"这是bug"——但正式版中他们会意识到"这是它在动"

种子3：孢子平台感知buff——第一次"身体变化"

Demo中，听完冥想者的文本后获得"感知"buff（命中率+5%）。

正式版中，这个buff的真实来源是： 玩家接触了一个低活跃度的觉醒节点，网络向玩家传递了微弱的感知能力。在Demo中它只是一个buff图标——在正式版中，玩家会意识到"我当时已经开始被渗透了，只是不知道"。

Demo中的埋设： 获得这个buff后，玩家在后续战斗中，有极低概率（5%）看到异常战斗日志文本（例如：「你打中了它。它知道是你打的。」）。大多数玩家不会触发。触发的玩家会困惑，然后忘记。但在正式版中，他们会想起：“原来第一次就在了。”

种子4：同伴的反水——觉醒事件的"人工预演"

Demo中同伴在Boss战第二阶段被感染、反水、释放。这是觉醒事件的"人工演示"：

  • 同伴被感染 = 他的"同化度"在孢子云中激增
  • 同伴反水 = 他的觉醒事件（从人类视角切换为网络视角）
  • 同伴被释放 = 通过外力干预，将他从网络中拉回

正式版中，玩家自己将经历这个过程。 同伴的反水是让玩家在情感上理解"被改变"是什么感觉——不是获得力量，而是被其他东西借用你的身体。

种子5：Boss战斗日志——不可理解化的叙事训练

Demo中Boss战斗日志使用"想起了疼痛"、“忘记了防御"等表述。这是在训练玩家接受一种非人类的叙事语言。

正式版中，觉醒物品的描述、网络节点的文本、同化后的对话选项——全部使用这种语言。Demo是在"教"玩家一种阅读方式：“这不是bug，这是另一种说话方式。”

正式版展开路径：从Demo到完整游戏

Act 1：接触（Contact）—— 觉醒物品的引入

目标： 让玩家意识到"有些物品不一样”——不是通过教程，而是通过"不对"

内容：

  • 离开真菌森林后，晶核在背包中开始出现行为异常（描述变化、偶尔移动）
  • 玩家获得第二件觉醒物品（主线任务或支线探索）
  • 第一次异常事件发生——玩家打开背包，发现物品位置变了。可能是自己记错了？
  • 玩家开始观察。没有教程说"注意观察物品行为"。玩家只是开始觉得"需要检查背包"
  • 第一次含混请求出现——晶核的描述变成"它想更近一点"。玩家猜测：更近什么？然后发现它朝某个方向移动了。也许它想靠近那个？

Act 1的结束标志： 第一次觉醒事件触发。玩家从"有些不对劲"进入"规则改变了"。

Act 2：渗透（Infiltration）—— 与背包居民共处

目标： 让玩家与背包中的物品建立"关系"——一种不确定、不平等、但真实的关系

内容：

  • 觉醒物品数量增加，异常事件频率增加
  • 物品开始主动回应玩家的行为——战斗中靠近武器、探索时朝特定方向移动
  • 玩家形成自己的假设（“它好像喜欢火”）——并通过行动验证
  • 验证有时成功、有时失败、有时部分成功——因果是暧昧的
  • 同化度开始通过身体变化显现——长休时偶尔有闪回、底座阴影偶尔抖动
  • 某些同伴开始疏远玩家，某些同伴开始好奇

Act 2的结束标志： 同化度达到"融合"阶段。玩家能稳定地看到隐藏路径，能"听到"网络的声音。觉醒事件不再是"突然的规则改变"，而是"日常的一部分"。

Act 3：融合（Fusion）—— 最终选择

目标： 让玩家面对"我是谁"的根本问题——不是哲学问题，是身体问题

内容：

  • 同化度达到"节点"阶段。玩家的身体已经显著改变
  • 某些NPC完全拒绝与玩家互动（“你不是……不对。你不是。"）
  • 某些NPC完全接受玩家（“你终于听到了。我们一直在等你。"）
  • 觉醒物品不再"请求”——它们开始"告知”。晶核不再滚动，而是稳定地脉动，仿佛在说：“我在这里。我一直在这里。”
  • 最终选择出现：不是对话选项，而是行为选择——
    • 回归： 玩家主动将所有觉醒物品分离，放入背包四角，不接触。长休后，它们不再移动。描述变回正常。同化度不再增长。但玩家失去了网络视角。世界变回"正常"——但玩家知道，正常是假象。
    • 拥抱： 玩家将觉醒物品聚集在一起，形成最大的相邻区域。长休后，它们形成一个"共鸣核心"。玩家完全融入网络。游戏继续，但目标改变——不再是"拯救世界"，而是"成为网络的一部分，让世界以新的方式继续"。
    • 平衡： 玩家维持当前状态——不聚集，不分离。让物品保持它们自己的节奏。这是最难的路线，需要玩家在后续内容中持续管理这种"不稳定的平衡"。最终结局暧昧——不是胜利，也不是失败，是"继续"。

与现有世界观的契合度

巨兽之梦

  • 觉醒物品的行为 = 巨兽梦中的"碎片意识"在梦中的自主运动——它们不是道具，是梦中的居民
  • 含混请求 = 碎片意识没有人类语言，它们用行为和感觉来沟通——就像梦中的符号，你知道它意味着什么，但说不清楚
  • 异常事件 = 梦境中的"逻辑漏洞"——梦越深入，漏洞越多，直到梦境开始"提醒"你这不是现实
  • 同化 = 玩家的意识从"角色"逐渐融入"巨兽的视角"——你不是在玩游戏，你是在被梦接纳
  • 最终选择 = 玩家决定自己在这个梦中的"位置"——醒来（回归）、沉睡（拥抱）、或者半梦半醒（平衡）

2.5D纸片人

  • 背包网格 = 高维存在对三维世界的"采样计算"——每个格子是一个计算单元，物品在格子间的移动是"计算优化"
  • 异常事件 = “采样误差”——梦的逻辑不稳定，偶尔出现"不该出现的东西"
  • 同化后的感知 = 玩家的"认知窗口"扩大，能看到更多"帧"——那些隐藏的菌丝路径，一直就在那里，只是之前看不到

冰山原则

  • 表面：玩家在整理背包，获得奇怪但有用的物品
  • 第一层：物品在互相影响，位置有某种意义
  • 第二层：物品在"要求"玩家做什么，玩家与物品的关系不平等——而且玩家越来越不确定谁在主导
  • 第三层：玩家不是在"使用"物品——是在与一个更大的意识"共处"。背包是这个意识在玩家体内的"居住空间"。最终，玩家必须决定：是继续做主人，还是承认自己也只是客人。

下一步建议

  1. Demo开发：继续按现有Demo文档推进。但确保：

    • 晶核的数据结构预留BehaviorSetID和DescriptionPoolID
    • Boss战后长休时，如果玩家在Demo中获得了孢子平台buff，晶核有极低概率（5%）移动一格（大多数玩家不会触发，但数据结构就位）
    • 战斗异常日志文本已写入数据表，但只在Demo中极低频触发

  2. 数据表设计：接下来需要设计——

    • DT_AwakenedItemBehavior：物品行为配置表（移动方向、偏好目标、描述池、行为脚本引用）
    • DT_AnomalyEvent：异常事件配置表（类型、触发概率、文本池、视觉效果）
    • DT_ObliqueRequest：含混请求配置表（请求形式、可能的解读方向、行为反馈）

  3. 概率曲线设计：需要为异常事件和觉醒事件设计概率曲线——让"越来越近了"可感知，但"什么时候触发"不可预测。这需要测试和迭代。

这份v2设计文档放弃了精确数值和明确条件，将系统完全转化为行为驱动的叙事引擎。玩家不是在管理背包，是在与一群不可知的室友共处——而共处的过程，就是异化的过程。
